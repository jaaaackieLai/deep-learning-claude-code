## Best Practices

1. **Conciseness**: SKILL.md should be under 80 lines. Link to references for details.
2. **Progressive disclosure**: Essential info in SKILL.md, full details in references.
3. **Concrete examples**: Every code section should have runnable snippets.
4. **Accuracy**: File paths and line numbers must reference actual repository files.
5. **Audience awareness**: Write for deep learning researchers who have not read the paper.
6. **Paper-code alignment**: Always note discrepancies between paper claims and code reality.
7. **Troubleshooting**: Include at least 2-3 common issues with concrete solutions.
8. **Hardware requirements**: Document GPU, RAM, and disk needs for training and inference.
