# 命令說明

本文檔說明市集中各命令的用途與使用情境。

---

## 概述

命令是以斜線呼叫的工作流程，用於觸發特定代理或流程。可作為常見開發任務的快捷方式。

---

## /plan

**用途**：在撰寫程式碼前建立實作計畫

**呼叫代理**：`planner`

**使用時機**：
- 開始新功能
- 進行重大架構變更
- 進行複雜重構
- 多個檔案/元件將受影響
- 需求不明確或有歧義

**執行流程**：
1. 以清晰語言重述需求
2. 分解為階段和具體步驟
3. 識別元件間的依賴關係
4. 評估風險和阻礙
5. **等待您的確認後才繼續**

**範例**：
```
/plan 我需要在市場結算時加入即時通知功能
```

---

## /tdd

**用途**：執行測試驅動開發工作流程

**呼叫代理**：`tdd-guide`

**使用時機**：
- 實作新功能
- 新增函式/元件
- 修復 bug（先撰寫重現 bug 的測試）
- 重構現有程式碼

**執行流程**：
1. 搭建輸入/輸出的介面
2. 撰寫失敗測試（RED）
3. 驗證測試因正確原因失敗
4. 實作最小程式碼使測試通過（GREEN）
5. 在保持測試綠色狀態下重構
6. 檢查覆蓋率（目標：80%+）

**範例**：
```
/tdd 我需要一個計算市場流動性分數的函式
```

---

## /test-coverage

**用途**：分析測試覆蓋率並生成缺失的測試

**使用時機**：
- 實作功能後
- 發布或合併前
- 覆蓋率低於 80% 時

**執行流程**：
1. 執行帶覆蓋率報告的測試
2. 識別低於 80% 門檻的檔案
3. 分析未測試的程式碼路徑
4. 生成單元、整合和 E2E 測試
5. 驗證新測試通過
6. 顯示前後覆蓋率指標

---

## /update-codemaps

**用途**：掃描程式碼庫結構並更新架構文件

**使用時機**：
- 新增重大功能後
- 架構變更時
- 發布新版本前
- 定期文件維護

**執行流程**：
1. 掃描所有源碼檔案的匯入、匯出和依賴關係
2. 生成精簡的 codemap（架構、後端、前端、資料）
3. 計算與前一版本的差異百分比
4. 變更超過 30% 時請求確認
5. 加入時間戳記

---

## /update-docs

**用途**：從真實來源檔案同步文件

**使用時機**：
- 變更專案配置後
- 環境變數變更時
- 新成員入職前

**執行流程**：
1. 讀取 package.json scripts 並生成參考表
2. 讀取 .env.example 並記錄環境變數
3. 生成 CONTRIB.md（開發工作流程、腳本、環境設定）
4. 生成 RUNBOOK.md（部署、監控、回滾）
5. 識別過時文件（90 天以上未更新）

---

## /commit

**用途**：分析所有未提交的變更，按功能分組建立小而聚焦的 commit

**使用時機**：
- 完成涉及多個檔案的工作後
- 想要乾淨、有組織的 commit 歷史時
- 取代單一龐大的「WIP」commit

**執行流程**：
1. 調查所有變更（已暫存、未暫存、未追蹤）
2. 按類型分類為 commit 群組（docs、feat、fix、refactor、test、chore、style）
3. 呈現編號計畫表
4. 依序執行 commit，使用 conventional commit 格式
5. 顯示最終 git log

**注意事項**：
- 遵循 Tidy First：標註混合的結構性/行為性變更，但不拆分單一檔案的 diff
- 絕不使用 `git add -A` -- 只加入特定檔案
- 對可能包含 secrets 的檔案發出警告

---

## /merge-worktree

**用途**：將當前 worktree 分支合併至 main 並清理

**使用時機**：
- 在 git worktree 完成工作後
- 準備將 worktree 分支整合回 main 時

**執行流程**：
1. 確認處於 worktree 中（非 main working tree）
2. 提交未 commit 的變更（需要時呼叫 `/commit`）
3. 顯示待合併的 commit，請求確認
4. 以 `--no-ff` 合併至 main
5. 移除 worktree 並刪除分支
6. 報告最終狀態

**注意事項**：
- 發生合併衝突時停止並報告（不強制解決）
- 有未追蹤檔案時會詢問是否強制移除 worktree

---

## 建議工作流程

功能開發時，按以下順序組合命令：

1. `/plan` - 了解要建構什麼，取得確認
2. `/tdd` - 以測試優先的方法實作
3. `/test-coverage` - 驗證覆蓋率達到門檻
4. `/update-docs` - 保持文件同步
5. `/commit` - 建立乾淨、聚焦的 commit
